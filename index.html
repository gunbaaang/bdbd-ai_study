한 줄 아이디어에서 배포까지

아래 단계를 순서대로 따라가며 진행하세요. 프롬프트는 복사 버튼으로 바로 붙여넣을 수 있습니다.

1

기획 체험

20분
핵심 강조 구간

목적: "한 줄 아이디어"가 왜 그대로 개발에 못 들어가는지 체감시키기.

챗봇 바로가기

G Gemini 앱 C ChatGPT (무료) A Claude (무료)

진행 방식

  1. 스터디원 각자 한 줄 아이디어 작성 (예: "동네 스터디룸 예약 사이트 만들고 싶어")
  2. 위 챗봇 바로가기 중 하나에 접속해 아래 템플릿을 입력
✎ 수정됨 (자동저장)
  1. 결과 프롬프트에서 "핵심 기능이 로직을 포함하는가" 검토 → 없으면 "예약 취소 시 재고 복구", "포인트 적립" 같은 조건부 로직을 추가한 후 재요청
여기서 나온 결과물을 그대로 복사해서 2단계 프롬프트에 붙여넣습니다.
2

빌드

30분

세팅

🚀 Google AI Studio Build 바로가기
  1. aistudio.google.com 접속 → Google 계정 로그인 → Build 탭 진입
  2. 새 프로젝트 생성, 모델은 Flash 계열로 고정 (한도 절약)

프롬프트 입력

1단계에서 만든 기획서 전체를 붙여넣기

✅ 로직 강제 체크포인트 — 빌드 후 아래 중 하나가 동작하는지 확인

  • 데이터 등록 → 목록에 즉시 반영되는가
  • 조건에 따라 화면/버튼이 달라지는가 (예: 로그인 여부, 재고 여부)
  • 관리자 계정으로 보면 일반 사용자와 다른 화면이 나오는가

안 되어 있으면 추가 프롬프트:

✎ 수정됨 (자동저장)
3

테스트 & 디버깅

15분
실습 전 진행자가 미리 확인해둘 것: AI Studio Build는 오류 발생 시 에러 메시지를 그대로 복사해서 프롬프트 창에 붙여넣으면 AI가 원인 분석 후 수정 코드를 다시 생성하는 구조입니다. 별도 콘솔/터미널 지식 불필요.

자주 나는 오류 유형과 대응 (클릭해서 펼치기)

원인: Firebase 승인된 도메인 미등록

해결: 아래 프롬프트를 그대로 입력하고, 안내받은 Firebase 콘솔 경로로 이동해 도메인을 추가하세요.

로그인이 안 돼. Firebase에서 승인된 도메인을 어디서 추가하는지 알려줘

원인: DB 연동 승인 누락

Add Database and Auth를 다시 확인해줘, 데이터가 저장되지 않아

원인: 무료 할당량 초과

해결: 잠시 대기하거나 모델을 Flash-Lite로 변경합니다. (별도 프롬프트 불필요)

원인: 프롬프트 지시가 모호함

해결: 스크린샷을 첨부하고 아래 프롬프트를 참고해 구체적으로 요청하세요.

이 화면에서 [구체적 요소]가 [의도한 대로] 안 보여. 스크린샷 기준으로 수정해줘

실습 방법

진행자가 일부러 도메인 미등록 상태로 배포 전 로그인 테스트를 시켜서, 오류 → "에러 메시지 복사 → AI에게 그대로 붙여넣기 → 원인 파악 → 해결"의 사이클을 1회 이상 경험하게 합니다.

4

배포

13분
  1. 상단 Publish 클릭
  2. 처음 배포 시 Google Cloud Run 연결 → 신용카드 등록 없이 첫 두 개 앱 무료 배포
  3. 배포 완료 후 생성된 URL을 팀원들과 공유
  4. 실제 접속해서 기능이 배포 환경에서도 동작하는지 확인
    • 안 되면 3단계와 동일하게 "배포 후 [기능]이 안 돼, 원인이 뭐야?" 프롬프트로 대응 (도메인 승인 재확인이 가장 흔한 원인)
✎ 수정됨 (자동저장)
🎉 여기까지 완료하면 실습 종료입니다. 배포된 URL을 팀원들과 공유하고, 번외 페이지에서 AI별 결과 차이도 함께 확인해보세요.
